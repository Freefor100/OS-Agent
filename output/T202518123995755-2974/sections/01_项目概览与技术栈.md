## 第 1 章：项目概览与技术栈

## 结论摘要

1. **项目名称与定位**：Nighthawk OS 是一个使用 Rust 编写的教学/竞赛操作系统项目，由哈尔滨工业大学（深圳）团队开发。该项目**并非基于 ArceOS 或 rCore**，而是参考了 PhoenixOS、MinotaurOS 等多个开源项目，采用自主架构设计。

2. **内核类型**：采用**混合内核架构**，支持异步协程调度模型（基于 Rust async/await），实现 M:N 多线程调度。内核核心位于 `kernel/` 目录，库模块位于 `lib/` 目录。

3. **架构支持**：支持 **RISC-V 64**（`riscv64gc-unknown-none-elf`）和 **LoongArch 64**（`loongarch64-unknown-none`）两种架构，通过条件编译和模块化设计实现跨平台支持。

4. **核心特性**：
   - ✅ 已实现：异步协程调度、Copy-on-Write 内存管理、懒分配、多核支持（M:N 调度）
   - ✅ 已实现：虚拟文件系统（VFS）、FAT32/Ext4 文件系统、网络栈（基于 smoltcp）
   - ✅ 已实现：近 200 个系统调用、信号机制、共享内存通信
   - 🔸 部分实现：多核调度（代码中存在 `panic!("multi-core unsupported")` 占位）

5. **技术栈**：Rust nightly-2025-01-18，no_std 环境，使用 buddy_system_allocator 作为堆分配器，bitmap_allocator 管理物理帧。

## 核心子系统概览

### 内存管理（✅ 已实现）

**物理内存管理**：
- 实现位置：`lib/mm/src/frame.rs`（293 行）
- 使用 `BitAlloc1M` 位图分配器追踪空闲物理帧
- 支持批量分配（`build_batch`）和连续分配（`build_contiguous`）
- RAII 机制：`FrameTracker` 自动回收，`FrameDropper` 批量回收

**堆内存管理**：
- 实现位置：`lib/mm/src/heap.rs`（89 行）
- 使用 `buddy_system_allocator`（buddy 算法）
- 全局分配器：`HEAP_ALLOCATOR`（`NoIrqLockedHeap<32>`）
- 堆大小：由 `config::mm::KERNEL_HEAP_SIZE` 定义

**虚拟内存与分页**：
- 实现位置：`kernel/src/vm/page_table.rs`（586 行）、`kernel/src/vm/vm_area.rs`（1167 行）
- 支持 Sv39（RISC-V）和相应 LoongArch 页表格式
- **Copy-on-Write 机制**（✅ 已实现）：
  - `addr_space.rs:295`：`clone_cow()` 函数实现 fork 时的 CoW 映射
  - `vm_area.rs:611`：`handle_cow_fault()` 处理写时复制缺页异常
  - 代码证据：
  ```rust
  // kernel/src/vm/addr_space.rs:295
  pub fn clone_cow(&self) -> SysResult<Self> {
      let mut new_space = Self::build_user()?;
      // ... 遍历 VMA，将写权限降级为只读
      if vma.flags().contains(VmaFlags::PRIVATE) && pte.flags().contains(PteFlags::W) {
          pte.set_flags(new_flags); // 移除 W 位
      }
  }
  
  // kernel/src/vm/vm_area.rs:611
  fn handle_cow_fault(&mut self, fault_addr: VirtAddr, pte: &mut PageTableEntry) -> SysResult<()> {
      if Arc::strong_count(fault_page) > 1 {
          // 分配新页并复制内容
          let new_page = Page::build()?;
          new_page.copy_from_page(fault_page);
          // ... 更新 PTE 为可写
      }
  }
  ```

**懒分配（Lazy Allocation）**：
- 实现位置：`kernel/src/vm/vm_area.rs`
- 通过 VMA 的 `handler` 回调机制实现缺页时分配
- 代码证据：`vm_area.rs:591-605` 中 `handle_fault` 调用 `handler.unwrap()` 延迟分配
