## 第 11 章：网络子系统与协议栈

## 网络子系统架构（自研 vs 第三方库）

**结论：当前主分支未实现真正的网络协议栈，仅有桩接口。**

根据 `os/Cargo.toml` 的依赖分析，项目**未引入** `smoltcp`、`lwip` 或其他 TCP/IP 协议栈库：

```toml
[dependencies]
riscv = { git = "...", optional = true }
sbi-rt = { version = "0.0.2", features = ["legacy"], optional = true }
buddy_system_allocator = "0.6"
virtio-drivers = { version = "0.2.0" }
lwext4_rust = { path = "../lwext4_rust" }
# 无 smoltcp、无网络协议栈依赖
```

文档 `Docs/决赛文档/net.md` 提到 `feature/net` 分支曾尝试基于 smoltcp 实现网络功能，但该分支**未合并到当前主分支**。当前代码中搜索 `smoltcp` 无任何结果。

**当前实现方式**：
- Socket 通信通过 `SimpleSocket` 实现，实际底层是**管道（Pipe）模拟**
- 数据收发通过全局 `UDP_QUEUE` 队列模拟网络数据包
- 仅支持本地回环（Loopback），无真实网络协议处理

## Socket 接口与系统调用

项目定义了完整的 Socket 系统调用接口（`os/src/syscall/mod.rs`），但大部分为**桩函数**：

| 系统调用 | 状态 | 实现位置 |
|---------|------|---------|
| `sys_socket` | 伪实现 | `os/src/syscall/net.rs:25` |
| `sys_bind` | 桩函数 | `os/src/syscall/net.rs:60` |
| `sys_connect` | 桩函数 | `os/src/syscall/net.rs:255` |
| `sys_listen` | 桩函数 | `os/src/syscall/net.rs:247` |
| `sys_accept` | 桩函数 | `os/src/syscall/net.rs:262` |
| `sys_sendto` | 伪实现（仅 Loopback） | `os/src/syscall/net.rs:140` |
| `sys_recvfrom` | 伪实现（仅 Loopback） | `os/src/syscall/net.rs:186` |
| `sys_socketpair` | 伪实现 | `os/src/syscall/net.rs:281` |

### `sys_socket` 实现分析

```rust
// os/src/syscall/net.rs:25-56
pub fn sys_socket(_domain: u32, _type: u32, _protocol: u32) -> SyscallRet {
    let domain: SocketDomain = SocketDomain::from(_domain);
    let task = current_task().unwrap();
    let task_inner = task.inner_lock();
    let new_fd = task_inner.fd_table.alloc_fd()?;
    // 创建 SimpleSocket（实际是 Pipe 包装）
    task_inner.fd_table.set(
        new_fd,
        FileDescriptor::new(flags, FileClass::Abs(make_socket())),
    );
    Ok(new_fd)
}
```

### `SimpleSocket` 本质是管道

```rust
// os/src/fs/files/socket.rs:11-16
pub fn make_socket() -> Arc<dyn File> {
    let (read_end, write_end) = make_pipe();  // 实际创建管道
    Arc::new(SimpleSocket {
        read_end,
        write_end,
    })
}

pub struct SimpleSocket {
    read_end: Arc<Pipe>,
    write_end: Arc<Pipe>,
}
```

**关键发现**：`SimpleSocket` 的 `read()` 和 `write()` 方法直接委托给 `Pipe`，**无协议栈处理**。

### `sys_sendto` / `sys_recvfrom` 限制

```rust
// os/src/syscall/net.rs:149-168
pub fn sys_sendto(...) -> SyscallRet {
    // 强制检查目标地址必须为 127.0.0.1
    if sockaddr_in.addr != 0x100007f {  // 127.0.0.1
        debug!("sys_sendto only support addr=127.0.0.1 !");
    }
    // 数据写入全局队列
    UDP_QUEUE.try_lock().unwrap().push_back(vec);
    Ok(1)  // 始终返回 1
}
```

**桩函数检测**：
- `sys_bind`：对 `AF_UNIX` 直接返回 `Ok(0)`，对 `AF_INET` 仅检查地址为全 0 后返回 `Ok(0)`
- `sys_connect`：直接返回 `Ok(0)`，无实际连接逻辑
- `sys_listen` / `sys_accept`：直接返回 `Ok(0)`，无监听队列管理
- `sys_sendmsg`：返回 `Err(SysErrNo::ENOSYS)`，明确未实现

## 协议栈支持详情（TCP/UDP/IP/Ethernet）

**结论：不支持任何网络协议。**

| ICMP | ❌ 不支持 | 无 ping 相关代码 |
| DNS | ❌ 不支持 | 无域名解析代码 |

## 数据包收发流程追踪

**结论：无真实数据包收发流程。**

### 当前伪实现流程

```
用户调用 sys_sendto() 
  ↓
检查目标地址是否为 127.0.0.1（硬编码检查）
  ↓
将数据拷贝到全局 UDP_QUEUE（VecDeque<Vec<u8>>）
  ↓
返回 Ok(1)（始终成功）

用户调用 sys_recvfrom()
  ↓
从 UDP_QUEUE 弹出数据
  ↓
拷贝到用户缓冲区
  ↓
返回数据长度
```

### 缺失的关键组件

1. **无网卡中断处理**：`os/src/drivers/virtio/` 仅有 `blk.rs`（块设备）和 `pci.rs`（PCI 配置），**无 `net.rs`**
2. **无 DMA 描述符管理**：搜索 `DMA` 仅在 `lwext4_rust` 文件系统相关代码中出现，无网络 DMA
3. **无 Ring Buffer**：无网卡收发队列管理
4. **无协议解析**：无 Ethernet 帧、IP 包、TCP/UDP 段的解析代码

## 高级特性支持验证（零拷贝等）

### 零拷贝（Zero Copy）
**❌ 不支持**

- 未找到 DMA 共享内存管理机制
- `sys_sendto` / `sys_recvfrom` 均使用**内存拷贝**（`Vec::push` 和逐字节拷贝）
- 无 `mbuf` 或类似零拷贝缓冲区设计

```rust
// os/src/syscall/net.rs:174-182
for i in 0.._len {
    unsafe {
        let c = memory_set.get_data(_buf.byte_add(i));  // 逐字节拷贝
        vec.push(c);
    }
}
```

### 多队列（Multi-queue / RSS）
**❌ 不支持**

- 无网卡多队列配置代码
- 无中断亲和性设置
- 单 `UDP_QUEUE` 全局队列设计

### 功能限制声明

**必须明确**：

1. **仅在 QEMU 环境测试**：项目文档未提及任何真实物理网卡测试记录
2. **仅支持本地回环**：`sys_sendto` / `sys_recvfrom` 硬编码检查 `127.0.0.1`
3. **无真实网络通信能力**：无法与外部网络或其他主机通信
4. **文档提及但未见代码**：`Docs/决赛文档/net.md` 描述的 smoltcp 实现、`InterfaceManager`、`SocketSetManager`、`ListenTable` 等组件**不存在于当前代码库**

## 网卡驱动细节

**结论：无网络网卡驱动。**

### 现有驱动分析

`os/src/drivers/` 目录结构：

### PHY/MAC 层抽象
**❌ 不存在**

### 错误处理流程

由于无真实网络操作，**无网络错误码传递流程**。所有网络系统调用要么返回 `Ok(0)` / `Ok(1)`，要么返回 `ENOSYS`（功能未实现）。

## 总结

| 特性 | 状态 | 备注 |
|------|------|------|
| 协议栈 | ❌ 未实现 | 文档提及 smoltcp 但代码不存在 |

**最终结论**：TatlinOS 当前主分支**不具备网络通信能力**。`Docs/决赛文档/net.md` 描述的网络功能存在于 `feature/net` 分支但未合并。当前代码中的网络相关系统调用均为**桩函数或伪实现**，仅支持基于内存队列的本地回环通信，无法用于真实网络环境。
