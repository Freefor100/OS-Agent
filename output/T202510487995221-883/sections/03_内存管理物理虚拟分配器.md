## 第 3 章：内存管理（物理/虚拟/分配器）

### 物理内存管理实现

#### 物理页分配器：PageCache + Buddy System

本仓库采用 **PageCache（页缓存）+ Buddy System（伙伴系统）** 的两级物理内存管理架构。

**1. 全局页缓存 `PageCache`**

位于 `os/src/mm/frame_alloc/page_cache.rs`，实现了一个基于水位的批量分配器：

```rust
pub struct PageCache {
    free_list: Mutex<Vec<PhysPageNum>>, // 空闲页链表
    count: AtomicUsize,                 // 当前缓存页数
}

impl PageCache {
    const HIGH_WATERMARK: usize = 128; // 缓存页数上限
    const LOW_WATERMARK: usize = 32;   // 缓存页数下限
    const REFILL_BATCH: usize = 16;    // 每次补充页数
    const FLUSH_BATCH: usize = 64;     // 每次刷回页数
}
```

- **分配流程** (`alloc`)：优先从 `free_list` 弹出空闲页；若为空，则批量从 `HEAP_ALLOCATOR` 申请 `REFILL_BATCH` (16) 页。
- **释放流程** (`dealloc`)：页回收到 `free_list`；当页数超过 `HIGH_WATERMARK` (128) 时，批量刷回 `FLUSH_BATCH` (64) 页到堆分配器。

**2. 物理页追踪器 `FrameTracker`**

位于 `os/src/mm/frame_alloc/frame_tracker.rs`，采用 RAII 模式管理单页生命周期：

```rust
pub struct FrameTracker {
    pub ppn: PhysPageNum,
}

impl FrameTracker {
    pub fn alloc() -> Option<Arc<FrameTracker>> {
        let ppn = PAGE_CACHE.alloc()?;
        let ret = Some(Arc::new(FrameTracker::new(ppn)));
        ret
    }
}

impl Drop for FrameTracker {
    fn drop(&mut self) {
        PAGE_CACHE.dealloc(self.ppn);
    }
}
```

- **清零初始化**：构造函数中清空物理页内容 (`bytes_array_mut` 逐字节置 0)。
- **引用计数**：返回 `Arc<FrameTracker>`，支持多所有者（如共享内存、COW）。
- **自动回收**：析构时自动调用 `PAGE_CACHE.dealloc`。

**3. 连续页分配 `ContinuousPages`**

位于 `os/src/mm/heap_allocator.rs`，用于分配多页连续内存（如内核栈）：

```rust
pub struct ContinuousPages {
    base: usize,
    page_num: usize,
    layout: Layout,
}

impl ContinuousPages {
    pub fn new(page_num: usize) -> Option<Self> {
        let size = page_num * PAGE_SIZE;
        let layout = Layout::from_size_align(size, PAGE_SIZE)?;
        let ptr = HEAP_ALLOCATOR.lock().alloc(layout)?;
        // 验证页面对齐
        assert_eq!(ptr.as_ptr() as usize % PAGE_SIZE, 0);
        Some(Self { base: ptr.as_ptr() as usize, page_num: size, layout })
    }
}
```

### 地址空间布局（内核 vs 用户）

#### 内核地址空间

内核使用独立的全局 `MemorySet`（`KERNEL_SPACE`），位于 `os/src/mm/memory_set/mod.rs`：

```rust
pub static KERNEL_SPACE: Lazy<Mutex<MemorySetInner>> =
    Lazy::new(|| Mutex::new(MemorySetInner::new_kernel()));
```

**内核重映射**：`new_kernel()` 函数将内核代码段、数据段、BSS 段重新映射到高位地址（`KERNEL_ADDR_OFFSET`），实现位置无关执行。

#### 用户地址空间

每个进程拥有独立的 `MemorySetInner`，包含：
- **代码段 (Elf)**：从可执行文件加载
- **数据段 (Brk)**：堆区，支持 `brk` 系统调用动态扩展
- **栈 (Stack)**：用户栈，位于 `USER_STACK_TOP`
- **Mmap 区**：动态映射区，位于 `MMAP_TOP` 向下增长

**地址空间隔离**：
- 内核页表项设置 `USER=0`，用户态不可访问
- 用户页表项设置 `USER=1`，内核可通过 `KERNEL_SPACE` 访问

**证据引用**：
- 内核空间初始化：`os/src/mm/memory_set/inner.rs:50-80`
- 用户空间布局：`os/src/arch/riscv64/qemu/memory_layout.rs:35-45`

### 堆分配器解析

#### 全局堆分配器 `HEAP_ALLOCATOR`

位于 `os/src/mm/heap_allocator.rs`，使用 `buddy_system_allocator` crate：

```rust
#[global_allocator]
pub static HEAP_ALLOCATOR: LockedHeap = LockedHeap::empty();

pub fn init_heap() {
    let start = end_of_kernel();
    let end = MEMORY_END;
    unsafe {
        HEAP_ALLOCATOR.lock().init(start, end - start);
    }
}
```

- **分配策略**：伙伴系统（Buddy System），支持 2 的幂次大小分配
- **初始化范围**：从内核结束地址 `end_of_kernel()` 到内存末尾 `MEMORY_END`

#### 堆管理 (`sys_brk`)

位于 `os/src/syscall/memory.rs:281-301`，支持**惰性分配**：

```rust
pub fn sys_brk(brk_addr: usize) -> SyscallRet {
    let task = current_task().unwrap();
    let mut task_inner = task.inner_lock();
    let old_addr = task_inner.user_heappoint;
    let grow_size: isize = (brk_addr - old_addr) as isize;

    let process = task.process.inner_lock();
    let memory_set = process.get_locked_memory_set();
    let ret = memory_set.get_mut().grow(grow_size, old_addr, task_inner.user_heapbottom);
    task_inner.user_heappoint = ret;
    Ok(ret)
}
```

**惰性分配机制**：
- `grow()` 函数（`os/src/mm/memory_set/inner.rs:150-185`）仅调整 `MapArea` 的 `vpn_range`，**不立即分配物理页**
- 实际物理页在首次访问时通过 `lazy_page_fault` 分配

**证据引用**：
- `sys_brk` 实现：`os/src/syscall/memory.rs:281-301`
- `grow()` 惰性扩展：`os/src/mm/memory_set/inner.rs:150-185`

### 用户指针安全

本仓库**未实现**标准的 `UserInPtr`/`UserOutPtr` 类型或 `verify_area`/`check_region` 函数。

**替代方案**：使用 `UserBuffer` 进行用户空间访问：

```rust
pub struct UserBuffer {
    pub buffers: Vec<&'static mut [u8]>,
    readonly: bool,
}

impl UserBuffer {
    pub fn new_safe(memory_set: &MemorySet, ptr: *const u8, len: usize, readonly: bool) -> Self {
        let buf = memory_set.safe_translated_byte_buffer(ptr, len, readonly);
        Self { buffers: buf, readonly: readonly }
    }
}
```

**安全机制**：
- `safe_translated_byte_buffer`（`os/src/mm/user_buffer.rs:45-70`）：在翻译用户指针时，若页不存在则触发 `lazy_page_fault` 分配
- **无显式边界检查**：依赖页表机制，非法地址会触发 Page Fault

**缺陷**：
- 未验证指针是否越界到内核空间
- 未检查跨页访问的连续性

**证据引用**：
- `UserBuffer` 实现：`os/src/mm/user_buffer.rs:75-150`
- 安全翻译：`os/src/mm/user_buffer.rs:45-70`

#### 完整调用链

```
            ├─> mmap_read_page_fault / mmap_write_page_fault (os/src/mm/page_fault_handler.rs:63-90)
            └─> lazy_page_fault (os/src/mm/page_fault_handler.rs:93-96)
                 └─> MapArea::map_one (os/src/mm/map_area.rs:100-130)
                      └─> PageTable::map (os/src/arch/riscv64/qemu/page_table.rs:170-210)
```

**处理逻辑**（`os/src/trap/mod.rs:90-130`）：

```rust
Trap::Exception(Exception::StorePageFault)
| Trap::Exception(Exception::LoadPageFault) => {
    let task = current_task().unwrap();
    let process = task.process.inner_lock();
    let memory_set = process.get_locked_memory_set();
    let is_write = cause == Trap::Exception(Exception::StorePageFault);
    
    // 1. 尝试懒加载
    ok = memory_set.lazy_page_fault(VirtAddr::from(stval).floor(), is_write);
    
    // 2. 若不是懒加载，尝试 COW（仅写故障）
    if !ok && is_write {
        ok = memory_set.cow_page_fault(VirtAddr::from(stval).floor());
    }
    
    // 3. 若都不是，发送 SIGSEGV 并终止进程
    if !ok {
        exit_current_and_run_next(-2);
    }
}
```

#### mmap 实现细节

**系统调用入口**（`os/src/syscall/memory.rs:21-85`）：

```rust
pub fn sys_mmap(addr: usize, len: usize, prot: u32, flags: u32, fd: usize, off: usize) -> SyscallRet {
    let map_perm: MapPermission = MmapProt::from_bits(prot).unwrap().into();
    let flags = MmapFlags::from_bits(flags)?;
    
    // 处理 MAP_FIXED
    if flags.contains(MmapFlags::MAP_FIXED) && addr == 0 {
        return Err(SysErrNo::EPERM);
    }
    
    // 匿名映射
    if fd == usize::MAX {
        if !flags.contains(MmapFlags::MAP_ANONYMOUS) {
            return Err(SysErrNo::EBADF);
        }
        return Ok(memory_set.mmap(addr, len, map_perm, flags, None, usize::MAX));
    }
    
    // 文件映射
    let file = task_inner.fd_table.get(fd).file()?;
    Ok(memory_set.mmap(addr, len, map_perm, flags, Some(file), off))
}
```

**标志位处理**（`os/src/syscall/options.rs:109-125`）：
- `MAP_SHARED` (1<<0)：修改同步到文件
- `MAP_PRIVATE` (1<<1)：私有映射
- `MAP_FIXED` (1<<4)：强制使用指定地址
- `MAP_ANONYMOUS` (1<<5)：匿名映射
- `MAP_STACK` (1<<17)：栈映射

**懒加载机制**：
- `mmap` 调用 `push_lazily` 创建 `MapArea`，不立即分配物理页
- 首次访问触发 `mmap_read_page_fault` 或 `mmap_write_page_fault`

**证据引用**：
- `sys_mmap` 实现：`os/src/syscall/memory.rs:21-85`

#### 2. COW 缺页异常处理

```rust
// os/src/arch/riscv64/qemu/page_table.rs:280-320
pub fn handle_cow_page_fault(&mut self, va: VirtAddr, vma: &mut MapArea) -> bool {
    let pte = self.find_valid_pte(va.floor())?;
    let pte_flags = pte.get_flags();
    
    // 检查 COW 标志
    if !pte_flags.contains(RVPTEFlags::COW) {
        return false;
    }
    
    // 若仅有一个引用，直接设置为可写
    let frame = vma.data_frames.get(&va.into()).unwrap();
    if Arc::strong_count(frame) == 1 {
        pte.set_flags(flags | RVPTEFlags::WD);
        return true;
    }
    
    // 否则：分配新页 + 复制内容
    let src = pte.get_ppn().bytes_array_mut();
    vma.unmap_one(self, va.into());
    vma.map_one(self, va.into());
    let dst = pte.get_ppn().bytes_array_mut();
    dst.copy_from_slice(src);
    
    true
}
```
