## 第 7 章：设备驱动与硬件抽象

### 驱动框架与设备发现

本项目实现了**基于 Feature 条件编译的静态驱动框架**，而非动态设备发现机制。驱动框架位于 `os/src/drivers/` 目录下，采用 Trait 抽象 + 条件编译选择具体实现的设计模式。

#### 驱动 Trait 定义

驱动框架的核心 Trait 定义在 `os/src/drivers/device.rs`：

```rust
pub trait BaseDriver: Send + Sync {
    fn device_type(&self) -> DeviceType;
}

pub trait BlockDriver: BaseDriver {
    fn num_blocks(&self) -> usize;
    fn block_size(&self) -> usize;
    fn read_block(&mut self, block_id: usize, buf: &mut [u8]) -> DevResult;
    fn write_block(&mut self, block_id: usize, buf: &[u8]) -> DevResult;
    fn flush(&mut self) -> DevResult;
}
```

设备类型枚举定义了四种设备：
- `Block`：块存储设备（磁盘）
- `Char`：字符设备（串口）
- `Net`：网络设备
- `Display`：图形显示设备

#### 设备发现机制：**无动态发现，采用静态配置**

经代码验证，本项目**未实现 Device Tree 解析或 PCI 动态扫描**。设备地址通过硬编码的内存映射常量定义：

**LoongArch64 平台**（`os/src/arch/loongarch64/qemu/memory_layout.rs`）：
```rust
#[cfg(not(feature = "board"))]
pub const UART_ADDR: usize = 0x8000_0000_1FE0_01E0;  // QEMU
#[cfg(feature = "board")]
pub const UART_ADDR: usize = 0x8000_0000_1FE2_0000;  // 上板

pub const MMIO: &[(usize, usize)] = &[
    (UART_ADDR - 0x8000_0000_0000_0000, 0x1000),  // UART0
    (0x100e_0000, 0x1000),                         // power off
    (0x2000_0000, 0x10000000),                     // PCI
];
```

**RISC-V 64 平台**（`os/src/arch/riscv64/qemu/memory_layout.rs`）：
```rust
pub const MMIO: &[(usize, usize)] = &[
    (0x0010_0000, 0x00_1000),  // VIRT_TEST
    (0x0010_1000, 0x00_1000),  // VIRT_RTC
    (0x1000_1000, 0x00_1000),  // Virtio Block
];
```

**PCI 配置空间访问**：虽然 `os/src/drivers/virtio/pci.rs` 实现了 `pci_config_read`/`pci_config_write` 函数，但这仅用于**硬编码设备 (0, DEVICE, 0)** 的配置，并非完整的 PCI 总线扫描。代码中存在遍历总线/设备/功能的循环，但实际使用的设备是硬编码的 `DEVICE = 1`。

#### 驱动注册与初始化

**未发现统一的驱动注册机制**。驱动初始化通过 `os/src/drivers/mod.rs` 中的条件编译在编译期确定：

```rust
cfg_if! {
    if #[cfg(feature = "ramdisk")] {
        type BlockDeviceImplInner = ramdisk::RamBlkDev;
        fn new_device_inner() -> BlockDeviceImplInner {
            ramdisk::RamBlkDev::new_ramdisk_device()
        }
    }
    else if #[cfg(all(feature = "riscv64", feature = "board"))] {
        type BlockDeviceImplInner = vf2::Vf2BlkDev;
        fn new_device_inner() -> BlockDeviceImplInner {
            vf2::Vf2BlkDev::new_vf2_device()
        }
    }
    else if #[cfg(all(feature = "riscv64", not(feature = "board")))] {
        type BlockDeviceImplInner = VirtIoBlkDev<VirtIoHalCMAImpl>;
        fn new_device_inner() -> BlockDeviceImplInner {
            unsafe { VirtIoBlkDev::new(&mut *(VIRTIO0 as *mut VirtIOHeader)) }
        }
    }
    else {
        // LA+qemu
        type BlockDeviceImplInner = VirtIoBlkDev2<VirtIoHalCMAImpl>;
        fn new_device_inner() -> BlockDeviceImplInner {
            VirtIoBlkDev2::<VirtIoHalCMAImpl>::new()
        }
    }
}
```

**关键发现**：`os/src/main.rs` 的 `rust_main()` 函数中**未调用任何驱动初始化函数**。块设备通过 `fs::init()` 中间接初始化，但字符设备（UART）在 MMU 启用前即可使用（通过 SBI 调用）。

| Feature 组合 | 块设备实现 | 说明 |
|-------------|-----------|------|
| `ramdisk` | `RamBlkDev` | 内存盘，镜像嵌入内核 `.S` 段 |

### 字符设备驱动（UART/Console）

#### MMU 启用前的串口访问

**RISC-V 64 平台**：通过 SBI 调用访问串口（`os/src/arch/riscv64/qemu/console.rs`）：

```rust
pub fn console_putchar(c: u8) {
    #[allow(deprecated)]
    sbi_rt::legacy::console_putchar(c as usize);
}

pub fn console_getchar() -> Option<u8> {
    #[allow(deprecated)]
    let c = sbi_rt::legacy::console_getchar();
    if c > 255 {
        None
    } else {
        Some(c as u8)
    }
}
```

**LoongArch64 平台**：直接访问 MMIO 寄存器（`os/src/arch/loongarch64/qemu/console.rs`）：

```rust
struct Uart16550 {
    base_address: usize,
}

impl Uart16550 {
    pub fn putchar(&mut self, c: u8) {
        let ptr = self.base_address as *mut u8;
        loop {
            unsafe {
                let c = ptr.add(5).read_volatile();
                if c & (1 << 5) != 0 {  // 检查 THRE 位
                    break;
                }
            }
        }
        unsafe {
            ptr.add(0).write_volatile(c);
        }
    }
}
```

#### MMU 启用后的地址切换

**关键发现**：本项目**未显式处理 MMU 前后串口地址切换**。UART 基地址 `UART_ADDR` 在 `memory_layout.rs` 中定义为**直接映射的虚拟地址**：

```rust
// LoongArch64
pub const KERNEL_ADDR_OFFSET: usize = 0x9000_0000_0000_0000;
pub const UART_ADDR: usize = 0x8000_0000_1FE0_01E0;  // 物理地址
// 实际访问时使用：UART_ADDR + KERNEL_ADDR_OFFSET (通过直接映射窗口)
```

LoongArch64 架构通过设置 `CRMD.DA` 位启用直接映射窗口，物理地址 `0x0000_xxxx_xxxx_xxxx` 自动映射到虚拟地址 `0x9000_xxxx_xxxx_xxxx`。因此，**UART 驱动始终使用虚拟地址访问**，无需显式切换。

#### Console 封装层

`os/src/console.rs` 提供统一的打印接口：

```rust
struct Stdout;

impl Write for Stdout {
    fn write_str(&mut self, s: &str) -> fmt::Result {
        for c in s.chars() {
            console_putchar(c as u8);
        }
        Ok(())
    }
}
```

**注意**：代码注释提到"在 LA 上板时，如果构造一个 `Mutex<Stdout>` 则会在 lock 时卡死"，因此当前实现**未使用互斥锁保护并发访问**。

### 块设备驱动（VirtIO-Blk 等）

本项目实现了三种块设备驱动：

#### 1. 内存盘驱动（RamBlkDev）

位置：`os/src/drivers/ramdisk/mod.rs`

```rust
global_asm!(include_str!("ramdisk.S"));

extern "C" {
    fn ramdisk_start();
    fn ramdisk_end();
}

impl BlockDriver for RamBlkDev {
    fn read_block(&mut self, block_id: usize, buf: &mut [u8]) -> DevResult {
        let block = unsafe {
            core::slice::from_raw_parts(
                ((ramdisk_start as usize) + (512 * block_id)) as *const u8,
                512,
            )
        };
        buf.copy_from_slice(block);
        Ok(())
    }
    
    fn write_block(&mut self, block_id: usize, buf: &[u8]) -> DevResult {
        let block = unsafe {
            core::slice::from_raw_parts_mut(
                ((ramdisk_start as usize) + (512 * block_id)) as *mut u8,
                512,
            )
        };
        block.copy_from_slice(buf);
        Ok(())
    }
}
```

**特点**：
- 镜像通过 `ramdisk.S` 汇编文件嵌入内核数据段
- 支持读写操作
- 适用于无外存的 QEMU 测试场景

#### 2. VisionFive 2 SD 卡驱动（Vf2BlkDev）

位置：`os/src/drivers/vf2/mod.rs`

```rust
pub const SDIO_BASE: usize = 0x16020000;
pub const KERNEL_SDIO_BASE: usize = SDIO_BASE + KERNEL_ADDR_OFFSET;

impl SDIo for SdIoImpl {
    fn read_reg_at(&self, offset: usize) -> u32 {
        let addr = (KERNEL_SDIO_BASE + offset) as *mut u32;
        unsafe { addr.read_volatile() }
    }
    fn write_reg_at(&mut self, offset: usize, val: u32) {
        let addr = (KERNEL_SDIO_BASE + offset) as *mut u32;
        unsafe { addr.write_volatile(val) }
    }
}

pub struct Vf2BlkDev(Mutex<Vf2SdDriver<SdIoImpl, SleepOpsImpl>>);
```

**特点**：
- 基于外部 crate `visionfive2-sd`
- 硬编码 SDIO 控制器基地址 `0x16020000`
- 实现 `SleepOps` 用于忙等待
- **仅支持 RISC-V 64 + board 组合**

#### 3. VirtIO 块设备驱动

**MMIO 版本**（RISC-V 64 QEMU）：`os/src/drivers/virtio/blk.rs`

```rust
pub struct VirtIoBlkDev<H: Hal> {
    inner: Mutex<VirtIOBlk<H, MmioTransport>>,
}

impl<H: Hal> VirtIoBlkDev<H> {
    pub fn new(header: &'static mut VirtIOHeader) -> Self {
        let transport: MmioTransport = unsafe { MmioTransport::new(header.into()).unwrap() };
        Self {
            inner: Mutex::new(VirtIOBlk::new(transport).expect("VirtIOBlk create failed")),
        }
    }
}
```

**PCI 版本**（LoongArch64 QEMU）：`os/src/drivers/virtio/pci.rs`

```rust
pub struct VirtIoBlkDev2<H: Hal> {
    inner: Mutex<VirtIOBlk<H, PciTransport>>,
}

const DEVICE: u8 = 1;  // 硬编码设备号

impl<H: Hal> VirtIoBlkDev2<H> {
    pub fn new() -> Self {
        unsafe {
            let root = 0x20000000 + KERNEL_ADDR_OFFSET;  // ECAM 基地址
            let mut root = PciRoot::new(root as *mut u8, Cam::Ecam);
            let func = DeviceFunction {
                bus: 0,
                device: DEVICE,
                function: 0,
            };
            // ... PCI 配置空间初始化 ...
            let mut transport = PciTransport::new::<H>(&mut root, func).unwrap();
            Self {
                inner: Mutex::new(VirtIOBlk::new(transport).unwrap()),
            }
        }
    }
}
```

**VirtIO HAL 实现**：`os/src/drivers/virtio/mod.rs`

```rust
pub struct VirtIoHalCMAImpl;

impl Hal for VirtIoHalCMAImpl {
    fn dma_alloc(pages: usize) -> usize {
        match cma_alloc(pages) {
            Some(addr) => addr.0,
            None => 0,
        }
    }

    fn dma_dealloc(pa: usize, pages: usize) -> i32 {
        cma_dealloc(PhysAddr(pa), pages);
        0
    }

    fn phys_to_virt(addr: usize) -> usize {
        KernelAddr::from(PhysAddr::from(addr)).0
    }
    
    // share/unshare 实现 DMA 缓冲区的同步
}
```

**关键发现**：
- 使用 `virtio-drivers` crate（版本 0.2.0）
- 实现 CMA（Contiguous Memory Allocator）用于 DMA 缓冲区分配
- PCI 版本硬编码设备号为 1，非动态枚举

### 目标平台适配情况

#### 支持的平台

| 架构 | 平台 | Feature 组合 | 块设备 | UART |
|------|------|-------------|--------|------|
| RISC-V 64 | QEMU virt | `riscv64 + !board` | VirtIO-MMIO | SBI |
| RISC-V 64 | VisionFive 2 | `riscv64 + board` | SD 卡 | SBI |
| LoongArch 64 | QEMU virt | `loongarch64 + !board` | VirtIO-PCI | MMIO |
| LoongArch 64 | 未知开发板 | `loongarch64 + board` | 未实现 | MMIO |

#### 平台适配机制

**内存布局差异**：

| 常量 | RISC-V 64 | LoongArch 64 |
|------|-----------|--------------|
| `PHYSICAL_MEMORY_START` | `0x8000_0000` | `0` (QEMU) / `0x90000000` (board) |
| `KERNEL_ADDR_OFFSET` | `0xffff_ffc0_0000_0000` | `0x9000_0000_0000_0000` |
| `UART 地址` | SBI 调用 | `0x1FE0_01E0` (QEMU) / `0x1FE2_0000` (board) |

**启动流程差异**：

- **RISC-V 64**：通过 SBI 调用访问固件服务（串口、关机等）
- **LoongArch 64**：直接访问 MMIO 寄存器，需要手动设置直接映射窗口

#### 未实现的板级支持

**LoongArch 64 + board** 组合在 `os/src/drivers/mod.rs` 中**无对应块设备实现**，会落入 `else` 分支导致编译错误（如果启用该组合）。

### 总结

本项目的设备驱动框架呈现以下特点：

1. **静态配置优于动态发现**：所有设备地址硬编码，无 Device Tree 或 ACPI 解析
2. **条件编译实现组件化**：通过 Cargo Features 选择驱动实现，编译期确定
3. **块设备驱动完善**：支持内存盘、SD 卡、VirtIO 三种后端
4. **字符设备依赖架构**：RISC-V 用 SBI，LoongArch 用 MMIO
5. **网络功能缺失**：文档设计完整，但代码未实现
6. **中断处理简化**：仅支持定时器中断，无外部中断管理
7. **平台适配有限**：LoongArch 上板模式缺少块设备支持

**反向证据声明**：
- 未找到 Device Tree 解析代码（`.dtb` 文件加载或解析）
- 未找到 smoltcp 网络协议栈集成代码
- 未找到 PLIC/APIC 中断控制器驱动
- 未找到 GPU、USB、音频等外设驱动
- 网络系统调用经代码验证为桩实现（仅操作内存队列，无真实网络 I/O）
