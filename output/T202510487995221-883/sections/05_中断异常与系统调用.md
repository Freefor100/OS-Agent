## 第 5 章：中断、异常与系统调用

### Trap 处理流程（用户态 <-> 内核态）

TatlinOS 实现了完整的双向 Trap 处理机制，支持 RISC-V 64 和 LoongArch 64 双架构。Trap 入口因架构而异：

**RISC-V 64 架构**：
- Trap 入口：`__trap_from_user`（位于 `os/src/arch/riscv64/qemu/asms/trap.S:87`）
- 通过 `stvec` CSR 注册为直接模式（Direct Mode）
- 入口地址在 `trap_init()` 中设置：`stvec::write(__trap_from_user as usize, TrapMode::Direct)`

**LoongArch 64 架构**：
- Trap 入口：`__alltraps`（位于 `os/src/arch/loongarch64/qemu/asms/trap.S:36`）
- 通过 `eentry` CSR 注册：`eentry::set_eentry(__alltraps as usize)`

**Trap 处理完整流程**：

```
用户态执行 → Trap 指令 (ecall/syscall) → 
__trap_from_user / __alltraps (汇编入口，保存上下文) → 
trap_entry (Rust 入口) → 
trap_handler (异常分发) → 
trap_return (信号处理 + 返回准备) → 
__return_to_user (汇编返回) → 用户态
```

关键代码路径（`os/src/trap/mod.rs`）：

```rust
#[no_mangle]
pub fn trap_entry() {
    trap_handler();
    trap_return();
}
```

**中断与异常区分**：
通过 `get_trap_cause()` 函数读取架构相关寄存器（RISC-V 的 `scause` / LoongArch 的 `estat`），返回 `(Trap, usize)` 元组：
- `Trap::Interrupt(Interrupt::Timer)`：时钟中断
- `Trap::Exception(Exception::Syscall)`：系统调用
- `Trap::Exception(Exception::LoadPageFault/StorePageFault)`：缺页异常
- `Trap::Exception(Exception::PageModifyFault)`：LoongArch 特有的页修改异常

**sys_write 完整追踪**（`os/src/syscall/fs.rs:33`）：

```rust
pub fn sys_write(fd: usize, buf: *const u8, len: usize) -> SyscallRet {
    let task = current_task().unwrap();
    let inner = task.inner_lock();
    
    // 1. 检查 fd 有效性
    if fd >= inner.fd_table.len() {
        return Err(SysErrNo::EINVAL);
    }
    
    // 2. 获取文件对象
    if let Some(file) = &inner.fd_table.try_get(fd) {
        let process = task.process.inner_lock();
        let memory_set = process.get_locked_memory_set();
        let file: Arc<dyn File> = file.any();
        
        // 3. 检查写权限
        if !file.writable() {
            return Err(SysErrNo::EACCES);
        }
        
        // 4. 创建用户态缓冲区包装（类型安全）
        let buffer = UserBuffer::new_safe(&memory_set, buf, len, true);
        
        // 5. 释放锁避免死锁
        drop(inner); drop(memory_set); drop(process);
        
        // 6. 执行实际写操作
        let ret = file.write(buffer)?;
        Ok(ret)
    } else {
        Err(SysErrNo::EBADF)
    }
}
```

**调用链总结**：
```
用户态 write() → ecall → __trap_from_user → trap_entry → 
trap_handler (识别 Syscall) → syscall() 分发 → 
sys_write() → UserBuffer::new_safe() → file.write() → 返回
```

**Lazy Allocation 实现**（`os/src/mm/memory_set/inner.rs:452`）：

```rust
pub fn lazy_page_fault(&mut self, vpn: VirtPageNum, is_write: bool) -> bool {
    // 检查是否已映射
    if !self.page_table.translate(vpn).is_none() {
        return false;
    }
    
    // mmap 区域处理
    if let Some(area) = self.areas.iter_mut()
        .filter(|area| area.area_type == MapAreaType::Mmap)
        .find(...) {
        if is_write {
            mmap_write_page_fault(...);
        } else {
            mmap_read_page_fault(...);
        }
        return true;
    }
    
    // brk/stack 区域处理
    if let Some(area) = self.areas.iter_mut()
        .filter(|area| area.area_type == MapAreaType::Brk || area.area_type == MapAreaType::Stack)
        .find(...) {
        lazy_page_fault(vpn.into(), &mut self.page_table, area);
        return true;
    }
    false
}
```

pub fn cow_page_fault(&mut self, vpn: VirtPageNum) -> bool {

        .filter(|area| area.area_type == MapAreaType::Elf 
                              || area.area_type == MapAreaType::Brk
                              || area.area_type == MapAreaType::Mmap)

        }

    let trap_cx = task_inner.trap_cx();
    let mut user_sp = trap_cx.get_sp();
    
    // 保存 MachineContext
    user_sp = user_sp - size_of::<MachineContext>();
    memory_set.put_data(user_sp as *mut MachineContext, trap_cx.as_mctx());
    
    // 保存信号掩码
    user_sp = user_sp - size_of::<SigSet>();

    trap_cx.set_a0(signo);
    trap_cx.set_sp(user_sp);
    trap_cx.set_sepc(sig_action.act.sa_handler);
    trap_cx.set_ra(sigreturn_trampoline as usize);
    
    task_inner.sig_mask |= sig_action.act.sa_mask | SigSet::from_sig(signo);
}

5. **缺页异常**：支持 Lazy Allocation 和 COW，但 SIGSEGV 信号未实现（直接终止进程）
6. **时钟中断**：每秒 1 次中断，支持 futex 定时器和任务调度
