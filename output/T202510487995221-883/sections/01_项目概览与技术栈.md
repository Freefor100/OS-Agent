## 第 1 章：项目概览与技术栈

## 结论摘要

1. **项目名称与来源**：本项目名为 **TatlinOS**（仓库名 `T202510487995221-883`），是基于 2024 年参赛作品 **TrustOS** 开发的操作系统内核，参赛队伍为黄羲成、常晔文、罗阳。

2. **技术定位**：这是一个使用 **Rust 2021** 编写的教学/竞赛向操作系统内核，支持 **RISC-V 64** 和 **LoongArch 64** 双架构，基于裸机环境（无标准库，`#![no_std]`）运行。

3. **核心框架**：项目采用模块化设计，包含内存管理（`mm/`）、进程管理（`task/`）、文件系统（`fs/`）、系统调用（`syscall/`）、中断处理（`trap/`）等核心子系统。文件系统基于 **lwext4**（C 语言实现的 ext2/3/4 库）通过 Rust FFI 封装提供 ext4 支持。

4. **构建系统**：使用 **Cargo + Makefile** 混合构建，通过 `make TARGET_ARCH=<arch>` 指定目标架构，支持 QEMU 仿真和真实硬件（"上板"）两种运行模式。

5. **功能实现状态**：已实现基础的分页内存管理（SV39/LoongArch TLB）、进程/线程管理（FIFO 调度）、ext4 文件系统、基础网络 socket 接口（基于 pipe 模拟，无真实网络栈）。部分系统调用为桩函数（如 `sys_sched_setscheduler` 返回 `Ok(0)` 无实际逻辑）。

pub fn rust_main(hartid: usize) -> ! {
    if FIRST_HART.load(Ordering::SeqCst) == true {
        clear_bss();
        trap::init();      // 中断初始化
        mm::init();        // 内存管理初始化
        logger::init();
        task::init();      // 任务管理初始化
        fs::init();        // 文件系统初始化
        task::add_initproc();
        arch::trap_interface::enable_timer_interrupt();
        timer::rtc_init();

        trap::init();
        mm::activate_kernel_space();
    }
    task::run_tasks();     // 启动调度器
    panic!("Unreachable in rust_main!");
}

## 核心子系统概览

### 内存管理（`os/src/mm/`）

**已实现功能**：

1. **分页机制**：
   - **RISC-V**: SV39 三级页表（`os/src/arch/riscv64/qemu/page_table.rs:465L`）
   - **LoongArch**: TLB + 页表（`os/src/arch/loongarch64/qemu/page_table.rs:653L`）
   - 支持内核/用户地址空间隔离（内核空间起始地址 `0xffff_ffff_c000_0000`）

2. **写时复制（CoW）**：
   - 页表项定义 `COW` 标志位（`LAPTEFlags::COW = 1 << 9`）
   - 实现 `handle_cow_page_fault()` 处理 CoW 缺页（`os/src/arch/loongarch64/qemu/page_table.rs:378`）
   - **验证代码**：
     ```rust
     // os/src/arch/loongarch64/qemu/page_table.rs:378
     pub fn handle_cow_page_fault(&mut self, va: VirtAddr, vma: &mut MapArea) -> bool {
         if !pte_flags.contains(LAPTEFlags::COW) {
             return false; // 非 CoW 页错误
         }
         // 分配新物理页，复制内容，更新页表为可写
         flags.remove(LAPTEFlags::COW);
         // ...
     }
     ```

3. **Lazy Allocation**：
   - 提供 `lazy_insert_framed_area()` 接口（`os/src/mm/memory_set/inner.rs:52`）
   - 页错误处理中支持按需分配（`os/src/mm/page_fault_handler.rs`）

4. **物理帧分配**：
   - 使用 `buddy_system_allocator` 伙伴系统（`os/src/mm/frame_alloc/mod.rs`）
   - 支持帧追踪器 `FrameTracker`

**未见实现**：
- 未找到内存压缩、交换分区（swap）相关代码

### 进程管理（`os/src/task/`）

**已实现功能**：

1. **任务结构**：
   - `TaskControlBlock`（TCB）：包含任务状态、上下文、文件描述符表、信号处理等
   - `Process`：进程级资源（地址空间、文件表、信号表）
   - 支持多线程（多 TCB 共享同一 `Process`）

2. **调度器**：
   - **FIFO 就绪队列**：`READY_QUEUE: VecDeque<Weak<TaskControlBlock>>`（`os/src/task/manager.rs:16`）
   - **调度策略**：简单时间片轮转（`run_tasks()` 循环取队首任务）
   - **多核支持**：每核独立 `Processor` 结构（`PROCESSORS: [Processor; HART_NUM]`）

3. **系统调用支持**：
   - 进程创建：`sys_clone()`（`os/src/syscall/process/clone.rs:118L`）
   - 进程执行：`sys_execve()`（`os/src/syscall/process/execve.rs:212L`）
   - 进程退出：`sys_exit()`、`sys_wait4()`
   - 线程同步：`sys_futex()`（`os/src/task/futex.rs:316L`）

**桩函数/未实现**：
- `sys_sched_setscheduler()`：仅打印警告并返回 `Ok(0)`（`os/src/syscall/process/others.rs:151`）
- `sys_sched_getscheduler()`：硬编码返回 `0`（`SCHED_OTHER`）（`os/src/syscall/process/others.rs:161`）
- **结论**：仅支持 FIFO 时间片调度，**未实现** CFS、优先级调度等高级调度算法

3. **挂载机制**：

   - 支持根目录挂载 ext4

**未见实现**：
- 未找到 FAT32、ext2/ext3（仅 ext4）、tmpfs 等其他文件系统实现

### 网络（`os/src/syscall/net.rs`）

**实现状态**：**仅有接口，无真实网络栈**

1. **Socket 实现**：
   - `sys_socket()`：创建基于 pipe 的伪 socket（`os/src/fs/files/socket.rs:11`）
   - `SimpleSocket`：内部使用两个 pipe 模拟双向通信
   - **验证代码**：
     ```rust
     // os/src/fs/files/socket.rs:11

         let (read_end, write_end) = make_pipe();
         Arc::new(SimpleSocket { read_end, write_end })
     }
     ```

2. **网络系统调用**：
   - `sys_bind()`、`sys_connect()`、`sys_listen()`、`sys_accept()`：均返回 `Ok(0)` 无实际逻辑
   - `sys_sendto()` / `sys_recvfrom()`：使用全局队列 `UDP_QUEUE` 模拟本地回环（`os/src/syscall/net.rs:23`）
   - **明确标注**：`sys_sendmsg()` 返回 `Err(SysErrNo::ENOSYS)`（未实现系统调用）

3. **未见实现**：
   - **无真实网络协议栈**：未集成 smoltcp、lwIP 等 TCP/IP 栈
   - **无网卡驱动**：未找到 VirtIO-net 或其他网卡驱动代码
   - **结论**：网络功能仅为桩实现，**不支持真实网络通信**
