# 关于「结论冲突」列的假阳性

初版表格中 `agreement=结论冲突` 对 **single_choice / multi_choice** 采用 **字符串全等** 判定。许多题目两侧 **选项语义相同**，仅差：

- 一侧写 **`A. xxx`**，另一侧只写 **`xxx`**（无字母前缀）；
- 或 JSON 多选一侧为 **`["F. …"]`** 另一侧格式略异。

此类应视为 **一致（表述差异）**，**不是**事实冲突。以本章叙述为准时，请优先阅读 `kernel_verification.md` 中 **仅对实质三态/多选语义分歧** 的裁定；未列入该文件的 `结论冲突` 行多为格式差异。

后续若自动化，可对 `single_choice` 在比较前剥离 `^[A-Z]\.\s*` 再比。
