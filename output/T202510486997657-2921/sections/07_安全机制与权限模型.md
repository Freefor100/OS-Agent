### Q07_012 UID/GID 字段是否在 syscall 路径上真实执行权限检查？（搜索 check_perm/inode_permission；若只有字段无检查链须标注「仅有定义但未强制执行 🔸」；给检查链证据或写「字段存在但无检查链」）

UID/GID 字段在 syscall 路径上真实执行权限检查，证据链完整：

1. sys_setuid/sys_setgid (kernel/sys/syscall_handler.cc:2765-2795):
   - sys_setgid: 检查 p->_euid != 0 && gid != p->_egid && gid != p->_gid 时返回 -EPERM
   - sys_setuid: 非特权用户检查 uid != p->_uid && uid != p->_euid 时返回 -EPERM

2. sys_faccessat (kernel/sys/syscall_handler.cc:5056-5290):
   - 获取 current_proc->_uid 和 current_proc->_gid
   - 目录权限检查：check_uid == dir_uid 用 owner_perms，check_gid == dir_gid 用 group_perms，否则用 other_perms
   - 文件权限检查：current_uid == file_uid 用 owner_perms，current_gid == file_gid 用 group_perms，否则用 other_perms
   - 缺少权限时返回 -EACCES

3. sys_readlinkat (kernel/sys/syscall_handler.cc:3470-3510):
   - 路径遍历中调用 vfs_mode_get/vfs_owner_get 获取目录权限和所有者
   - proc_uid == dir_uid 检查 S_IXUSR，proc_gid == dir_gid 检查 S_IXGRP，否则检查 S_IXOTH
   - 无搜索权限时返回 SYS_EACCES

结论：UID/GID 字段不仅存在，且在多个 syscall 中强制执行权限检查，非桩函数。
