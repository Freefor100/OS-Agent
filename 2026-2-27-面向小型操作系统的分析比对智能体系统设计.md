## 项目名称

面向小型操作系统的分析比对智能体系统设计

## 项目描述

自 2021 年全国大学生操作系统比赛（OS Kernel 赛道）开展以来，参赛选手基于 RISC-V 等架构开发了大量设计精巧、风格各异的小型操作系统（如各种 RCore/UCore 的变体、独立实现的微内核等）。随着比赛步入第六年（2026年），这些开源代码仓（Repo）已经积累了巨大的数量。如何高效地对新提交的设计进行自动化评价、理解其技术演进路径，并准确区分新设计与历史存量设计之间的技术差异（查重与创新点分析），已成为组委会和评审专家面临的新挑战。

本项目旨在利用当前流行的大语言模型（LLM）工具与智能体（Agent）技术，构建一套专门面向小型操作系统源码的智能分析系统。该系统需具备深度语义理解能力，能够超越简单的文本对比，从内核结构、函数调用逻辑、开发历史等维度，自动生成对项目的结构化描述，并实现与历届比赛作品的深度比对。

## 特征

- **多维感知**：不仅分析代码，还需分析 Git 提交历史、Makefile/CMake 构建配置。
- **语义比对**：利用向量数据库或 RAG（检索增强生成）技术，在语义层面而非仅在字符层面识别代码相似度。
- **可视化输出**：能够生成直观的内核调用图（Call Graph）并提供对比报告。

## 参考实现

- AutoGPT / LangChain / MetaGPT (主流 Agent 框架)
- LLM 模型: GPT-4, Claude 3.5, 或国产优秀大模型（通义千问、文心一言等），开源/免费模型优先。
- Doxygen / Graphviz (用于生成函数调用图的参考工具)
- Vector Databases (Milvus, Pinecone 等，用于存储历史作品指纹)

## 预期目标

### 注意：下面的内容是建议内容，不要求必须全部完成。选择本项目的同学也可与导师联系，提出自己的新想法，如导师认可，可加入预期目标。

### 第一题：项目自动化描述智能体

- **概况提取**：自动识别并描述项目的开发语言（如 Rust, C）、目标平台（如 K210, QEMU）、开发环境（如 Docker, Toolchain）及内核结构（如 Macrokernel, Microkernel）。
- **过程勾勒**：通过分析 Git Log 或提交频率，自动总结项目的主要开发阶段与时间线。
- **图谱生成**：自动静态分析源码，生成核心子系统（进程管理、内存管理、文件系统）的函数调用图（Call Graph）。

### 第二题：智能化查重与比对系统

- **相似性检索**：在给定的历史作品库中，快速检索出与目标项目相似度最高的 5 个候选作品。
- **差异化报告**：针对选出的 5 个作品，分别生成详细的比对报告。报告需明确指出：代码实现的重合部分、技术架构的创新点、以及逻辑实现的差异。

### 第三题：系统性能与准确性

- **准确率**：对已知“复刻/抄袭”案例的识别准确率需达到 90% 以上。
- **响应速度**：单个万行代码规模项目的完整分析报告生成时间应控制在 5 分钟以内。
